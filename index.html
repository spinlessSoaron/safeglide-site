What is safeGlide?

safeGlide is a personal toolkit for soaring pilots that analyses recorded flights for dangerous flying behaviour — situations that are low and slow and close to terrain, or where a landable field is no longer within glide range. It works on a single flight, or across an entire flying career.

🪂

Single-flight review

Load one IGC file and see the whole flight on an interactive map, with altitude / speed / danger plots and a table of flagged episodes and thermal climbs.

📈

Career trends

Aggregate the danger events from every flight you've logged and plot them as smoothed rate curves (events per flight-hour) against cumulative experience.

⛰️

Terrain & ridge intelligence

Uses SRTM elevation tiles, landable-field databases, and a watershed-derived ridge map to judge terrain proximity and required glide ratio.

📱

Native apps

A SwiftUI GliderTrends app (macOS / iOS) reproduces the trend engine bit-for-bit, plus an in-flight GliderCockpit companion.

The danger model

Each flight is reconstructed from its IGC trace — speed, heading, climb rate, height above terrain — and run through an energy-aware, four-detector danger model. Rather than flagging raw altitude, it looks at the combination of total energy, proximity to the ground, and whether a landing option is still reachable. Detected episodes are reduced to discrete events, each carrying a type and a gravity:

caution warning critical

Event types group the reason: circling low, terrain proximity, and field-reach (the nearest landable field requires a glide ratio you may not have). Curves are separated by both type and gravity so you can see, for example, whether your "circling-low" warnings have become rarer with experience.

Per-flight summaries are cached on disk (keyed by file path, modification time and terrain flag), so re-running over a growing logbook only analyses the new flights.

User guide

The analysis tools are Python; the native apps are Swift. Here's how to get the Python tools running.

  1. Get the code & create a virtual environment

    From the project folder:

    python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt

    This installs the analysis stack — aerofiles (IGC parsing), numpy/scipy, geopy, folium/plotly for maps & charts, and PyQt6 for the desktop GUIs.

  2. Review a single flight

    Launch the single-flight viewer, then open any .igc file:

    python gui_app.py

    You get an interactive map of the track with flagged danger episodes highlighted, altitude / speed / danger-level plots, and a table of episodes and thermals. Terrain checks use local SRTM tiles and a SeeYou .cup field database.

  3. See your career trends

    Launch the trends analyser:

    python trends_app.py

    Then gather flights one of two ways:

    • From WeGlide — with your WeGlide API key, your own flights are downloaded into weglide_downloads/ and analysed automatically.
    • From local files — every *.igc whose filename contains a name you type (across the project folder, the download folder and ~/Downloads) is analysed.

    The result is a set of smoothed event-rate curves over your cumulative flight hours, with an adjustable smoothing window.

  4. (Optional) Run the native app

    The SwiftUI port lives in GliderTrends/ and builds with Swift Package Manager on macOS:

    cd GliderTrends
swift build
swift run GliderTrendsApp

    Its numerical engine is verified bit-for-bit against the Python reference via golden fixtures.

Data you'll need: IGC flight logs, SRTM .hgt elevation tiles for terrain proximity (downloaded on demand by the Swift app), and a SeeYou .cup file of landable fields. A WeGlide API key is only required for the WeGlide download path — note that WeGlide only lets the key's owner download their own flights.
